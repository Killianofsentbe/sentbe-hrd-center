S4 신설 배경 (04.06 Align): 기존 S3가 너무 넓어 평가·징계 등 극소수 접근 자료까지 포함. S3를 세분화하여 S4를 분리 신설. 이전 S1·S2 자료 중 일반 사내 공유 가능한 것은 S1로 통합, 기존 S3 중 HR팀 공유 가능한 것은 S2로 이동.

6
Do / Don't 요약
✅ DO
  • 폴더명에 _ 사용 (띄어쓰기 금지)
  • 번호는 두 자리 고정 (01, 02 …)
  • 파일명에 날짜·버전·작성자 포함
  • 보안 레벨이 다르면 폴더 분리
  • 완료 자료는 99_보관으로 이동
  • 기능 기준으로 폴더 생성
  • 폴더 깊이 4단계 이내 유지
❌ DON'T
  • 최종, 최종2, 진짜최종, new, last
  • 새폴더, 기타, temp, misc, 정리필요
  • 사람 이름 중심 폴더 (민수자료, jane work)
  • 같은 자료 여러 폴더에 중복 저장
  • 보안 다른 자료 같은 폴더에 혼합
  • 5단계 이상 깊은 중첩 습관화
  • 재무↔회계, HR↔인사 등 유사명 혼용
🔧
마이그레이션 진행 순서 (04.06 Align 후속)
  1. 저스틴 → 기존 공유 드라이브 전체 트리 구조 공유
    현재 폴더 목록 파악이 선행. 스크린샷 또는 텍스트 트리로 공유 요청.
  2. 킬리안 → 기능별 폴더 분류 초안 작성 + 담당자 매핑
    위 트리 기준으로 각 폴더가 어느 기능·보안 레벨에 해당하는지 초안 작성.
  3. 기능별 담당자에게 해당 폴더 권한 부여
    각 기능 폴더의 적정 담당자를 지정하고 S2/S3 수준 폴더에 권한 위임.
  4. 담당자가 신규 구조로 파일 이동
    이 퀵 가이드 규칙에 따라 각자 담당 폴더 정리. 일괄보다 우선순위 높은 폴더부터.
  5. 구조 검토 후 S4 대상 폴더 별도 권한 재설정
    평가·징계 등 S4 해당 자료는 이동 완료 후 저스틴·킬리안이 권한 최종 조정.
💡
기존 자료는 일괄 이동보다 우선순위 높은 폴더부터 점진적 정리. 이 가이드는 반기별 또는 필요 시 리뷰.
